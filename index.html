pickle

The pickle module is part of the Python standard library and is used to "pickle" or serialize arbitrary Python objects.

What is "serialization"?

Serialization is the process of translating data structures or object state into a format that can be stored or transmitted and reconstructed later.

Common serialization Formats

  • JSON
  • YAML
  • XML
Python pickle predates all these protocols

Why Pickles?

  • Pickles can serialize (almost) any Python object.
  • Pickles work with nested/self-referencing objects.
  • Pickles use an efficient binary format.
  1. scikit-learn
  2. Jupyter notebook
  3. code as data

Why Not Pickles?

  • Pickles are Python-only.
  • Pickles are not meant to be read/written by humans.
  • Pickles are not secure.

Honorable Mention

  • shelve
  • struct

shelve

  • Key-value store: keys are strings; values are arbitrary Python objects
  • Part of the Python standard library
  • Values are stored as pickles

struct

  • Used to read/create binary ("C") data types (structs).
  • e.g. reading/writing graphics files, controlling the terminal, reading/writing zipped files, making pickles.
